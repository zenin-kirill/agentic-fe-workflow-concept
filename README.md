# Концепт мультиагентного процесса разработки клиентских приложений на базе BDD и TDD методологий и LLM моделей

Концепт агентского процесса frontend-разработки в виде презентации, сформированный и проработанный на практике за полгода повседневной эксплуатации.

Презентация показывает, как превратить свободную сессию ИИ-ассистента в инженерный процесс с планированием, разделением ролей, обязательными проверками качества и явным участием пользователя в ключевых решениях. Причины именно такого подхода и выбора решений в рамках данного репозитория не рассматриваются.

## Идея

Обычный ИИ-ассистент часто сразу переходит к изменению кода. В этом подходе работа строится иначе:

- сначала анализ и план;
- затем проверка плана независимыми агентами;
- затем реализация единственным исполнителем;
- затем проверка результата;
- только после этого — фиксация изменений.

Переходы между этапами блокируются не инструкциями для модели, а средой исполнения: без обязательных проверок и явного согласия пользователя процесс не может перейти дальше.

## Динамика процесса

Процесс не является жёсткой линейной цепочкой. Оркестратор управляет им гибко:

- пропускает шаги, которые не требуются для реализации конкретной задачи;
- запускает независимые подзадачи параллельно;
- возвращает план на доработку, если проверяющие нашли замечания;
- возвращает реализацию исполнителю, если проверки качества не пройдены;
- формирует циклы доработки до тех пор, пока результат не станет приемлемым.

## Архитектура

Система состоит из трёх слоёв:

| Слой             | Назначение                                                  |
| ---------------- | ----------------------------------------------------------- |
| Навыки           | Правила, методологии и справочники                          |
| Агенты           | Аналитики, исполнитель, проверяющие и служебные роли        |
| Среда исполнения | Состояние процесса, события, контрольные точки и блокировки |
